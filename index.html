What does any of this mean?

This page has three parts: a plain-English dictionary of programming words, a note on how the keyword matching actually works, and a full reference of every template and modifier in the library — including what each one does and when you'd use it.

How matching works

There's no AI here — it's closer to a dictionary lookup. When you type something, the tool:

1. Splits your text into individual words.

2. Trims common endings (so "sorting", "sorted", and "sorts" all match "sort").

3. Checks each word against a fixed list of trigger words attached to each code template below.

4. Assembles the matching templates — plus any modifiers you typed, like "tests" or "cli" — into one Python file.
